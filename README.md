# ath-protocol
# Agent Trust Handshake Protocol
## 智能体互联网可信握手协议

---

## 🔍 项目概述
**ATH Protocol** 是面向智能体生态设计的**纯应用层轻量化安全通信协议**，独立于业务能力调用、工具执行逻辑，仅专注解决智能体跨平台/跨节点通信的**可信准入**核心问题。
作为智能体互联网的**安全底座**，协议具备轻量化、去中心化、无侵入、全平台兼容的核心特性。

---

## ⚠️ 行业痛点（协议解决的核心问题）
当前智能体通信面临四大安全隐患：
1. 无唯一身份标识，人机混淆、伪造风险高
2. 无统一通信发现机制，跨节点互联成本高
3. 无标准化可信认证流程，通信安全性无法保障
4. 无细粒度权限管控，越权访问、滥用风险突出

---

## 🆚 与 MCP 协议核心区别（官方定位）
| 协议 | 层级定位 | 核心职责 | 适用场景 |
| ---- | -------- | -------- | -------- |
| **MCP** | 业务层协议 | 规范智能体「能力调用、工具交互、业务执行」 | 智能体业务逻辑实现 |
| **ATH Protocol** | 安全底座协议 | 规范智能体「通信握手、身份核验、权限准入」 | 智能体通信安全保障 |

✅ **互补关系**：ATH 负责通信安全准入，MCP 负责上层业务调用，两者无缝兼容、协同工作。

---

## ✨ 核心能力
- **对象发现**：统一标准化智能体通信发现机制，降低跨生态互联成本
- **数字身份**：基于 W3C DID 标准，生成全局唯一、不可篡改的智能体身份
- **双维度认证**：AI 应用身份核验（必选）+ 用户代表授权核验（可选）
- **细粒度鉴权**：支持允许/受限/拒绝三级权限策略，遵循最小权限原则
- **跨生态互认**：兼容所有智能体平台、业务协议，实现全局身份互信
- **极致轻量化**：无底层改造、10 分钟快速集成、无业务侵入性

---

## 🎯 协议定位
- 智能体通信的 **安全门禁**
- 智能体互联网的 **基础安全设施**
- 通用型安全底座，可适配所有上层业务协议（含 MCP）
- 基于 HTTPS 标准传输，兼容现有互联网安全体系

from ath_sdk import AthClient

if __name__ == '__main__':
    # ======================
    # 1. 初始化智能体（配置唯一DID身份）
    # ======================
    agent = AthClient(agent_did="did:ath:official-agent-2025001")

    # ======================
    # 2. 执行对象发现（查找目标通信节点）
    # ======================
    agent.discover(target_url="https://api.ath-protocol.com")

    # ======================
    # 3. 双维度身份认证
    # ======================
    # AI 应用身份令牌（由 ATH 认证中心颁发）
    agent.authenticate(
        ai_token="ath-ai-credential-official-123456",
        user_token="user-oauth2-consent-token-789"  # 可选：用户授权凭证
    )

    # ======================
    # 4. 访问权限鉴权
    # ======================
    auth_result = agent.authorize()
    print("=" * 50)
    print("ATH 可信握手完成")
    print(f"鉴权状态: {auth_result['message']}")
    print(f"通信权限: {auth_result['data']['status']}")
    print("=" * 50)
